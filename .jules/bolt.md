## 2024-05-18 - Throttle MutationObserver in Vanilla JS
**Learning:** Pure vanilla JS Chrome extensions often rely on `MutationObserver` to watch for dynamically loaded DOM elements (like images or canvas changes) created by the main world page scripts. Calling somewhat expensive logic (like regex matching or state updates) synchronously inside a rapidly firing observer can block the main thread and lead to callback starvation or jank, especially on pages with continuous animations or rapid loading.
**Action:** When implementing continuous DOM observers (`MutationObserver`) in vanilla JS, prefer throttling execution by batching DOM reads using `requestAnimationFrame` combined with a boolean lock flag. This ensures the parsing logic runs at most once per frame, remaining highly efficient without blocking the UI thread.
